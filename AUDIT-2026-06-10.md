## Key Fixes Applied

### 1. Dashboard CI/install stability

`dashboard/package-lock.json` was out of sync with `package.json`, causing `npm ci` to fail. The lockfile was refreshed and now strict CI install passes.

### 2. Watch page keyboard source switching

`Watch.tsx` referenced `switchSource` before declaration inside the keyboard shortcut effect. It was converted to a `useCallback` declared before the effect, and source cycling now safely handles streams with zero sources.

### 3. ESLint configuration professionalized

The project uses pragmatic React patterns such as context files exporting both providers and hooks, plus localStorage hydration inside effects. ESLint rules were tuned to keep correctness checks strong without blocking on noisy false positives.

### 4. Bash detection/recording hardening

Cookie CLI arguments were moved from string-based expansion to Bash arrays in the critical yt-dlp/streamlink paths. This prevents word-splitting bugs and is safer if file paths ever contain spaces.

Affected scripts:

- `scripts/detect-stream.sh`
- `scripts/record-stream.sh`

### 5. Segment concat bug cleaned

The segment merge code used a bare redirection:

```bash
> "$concat_file"
```

This is legal Bash but poor hygiene and flagged by ShellCheck. It is now explicit:

```bash
: > "$concat_file"
```

### 6. Cookie file writing hardened

Workflow cookie preparation now uses `printf` instead of `echo`, preserving Netscape cookie rows more reliably. Cookie files are also chmodded to `600`.

Affected workflows:

- `.github/workflows/stream-recorder.yml`
- `.github/workflows/youtube-to-archive.yml`

### 7. Workflow cleanup made best-effort safe

The old-run cleanup step in `stream-recorder.yml` now skips safely if `GH_PAT` or `jq` is unavailable and uses stricter GitHub API curl flags. It will not block recording.

### 8. Production dashboard URL corrected

Discord/dashboard links now default to:

```text
https://muslim-lantern-archive.vercel.app/
```

instead of the GitHub Pages fallback.

### 9. New Quality Check workflow

Added:

```text
.github/workflows/quality-check.yml
```

It validates:

- Bash syntax
- ShellCheck error-level findings
- Workflow YAML parsing support via repo checks
- Dashboard `npm ci`
- Dashboard lint
- Dashboard production build
- Secret reference summary in the GitHub Actions job summary

### 10. Dashboard SystemHealth upgraded

`SystemHealth.tsx` now fetches live static repo data from raw GitHub:

- `stats.json`
- `cookie_timestamp.txt`

It now displays real archive state instead of hardcoded placeholder dates/statuses.

## Additional Phase 2 Enhancements

After the first pass, the following reliability/professional polish was added:

### Archive.org metadata branding

`upload_to_archive` now creates `tml-*` identifiers instead of generic `pmb-*` identifiers and writes branded Archive.org metadata:

- title = stream title + quality/part label
- creator = The Muslim Lantern / detected channel
- description = Muslim Lantern Stream Archive wording
- subjects = The Muslim Lantern, Islam, Dawah, live stream, archive

### Provider source counters in `stats.json`

`update-stats.sh` now maintains a `sources` object in `stats.json`:

```json
{
  "sources": {
    "archive": 0,
    "mega": 0,
    "pixel": 0,
    "gofile": 0
  }
}
```

Counts are incremented once per successful recording if at least one link for that provider was produced. This makes dashboard provider stats accurate and backend-generated instead of estimated client-side.

### Homepage metrics now prefer backend-generated stats

`Home.tsx` now fetches `stats.json` directly and uses backend-calculated totals for streams, hours, storage, and provider counts. It falls back to deriving approximate totals from parsed stream data only if `stats.json` is unavailable.

### Gallery provider filters expanded

Gallery filters now include Pixeldrain and Gofile provider filters in addition to Archive and MEGA.

### Dashboard feed merge order fixed

`data/recordings.json` updates now deduplicate while preserving newest-first order. The previous `unique_by(.video_id)` approach could reorder entries because jq sorts before uniquing.

## Final Status

The repo is in a better professional state after this pass. The critical validation suite passes locally, and the new quality workflow should help prevent future regressions.
